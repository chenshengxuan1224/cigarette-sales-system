# 香烟零售户销售系统实施计划

## 项目概述

创建一个香烟零售户每日销售管理系统，支持香烟品牌管理、入库登记、销售记录和多维度营业数据统计分析。

## 技术方案

- **前端**: 纯 HTML5 + CSS3 + JavaScript (Vanilla)
- **数据存储**: LocalStorage（本地持久化）
- **样式**: 现代简洁风格，使用 Flexbox 和 Grid 布局
- **图标**: 使用 Font Awesome CDN

## 功能需求

### 1. 品牌管理
- 添加香烟品牌（名称、规格、入库价格、销售价格）
- 编辑品牌信息
- 删除品牌
- 品牌列表展示
- 品牌库存管理

### 2. 销售记录
- 选择品牌快速录入销售数量
- 自动计算销售金额和利润
- 销售记录列表（支持按日期筛选）
- 删除销售记录
- 批量录入销售

### 3. 数据统计（多维度）

#### 3.1 实时仪表盘
- 今日销售额
- 今日销售量（条数）
- 今日利润
- 今日利润率

#### 3.2 按日期统计
- 选择日期范围统计
- 每日销售趋势图
- 周销售对比
- 月销售汇总

#### 3.3 按品牌统计
- 各品牌销售数量排名
- 各品牌销售金额占比
- 各品牌利润贡献分析
- 畅销品牌排行榜

#### 3.4 利润分析
- 总销售额
- 总成本（按入库价计算）
- 总利润
- 毛利率
- 利润率

#### 3.5 库存统计
- 当前库存总量
- 库存金额（按入库价计算）
- 库存预警（库存不足提醒）

### 4. 数据持久化
- 所有数据保存到 LocalStorage
- 刷新页面数据不丢失
- 数据导出功能（JSON 格式）

## 数据结构

### Brands（品牌）
```json
{
  "id": 1,
  "name": "中华",
  "spec": "软盒",
  "purchasePrice": 45,
  "salePrice": 50,
  "stock": 100
}
```

### Sales（销售记录）
```json
{
  "id": 1,
  "brandId": 1,
  "quantity": 5,
  "salePrice": 50,
  "purchasePrice": 45,
  "date": "2024-01-15",
  "timestamp": 1705305600000
}
```

## 风险与注意事项

- LocalStorage 存储容量有限（约 5MB），需提示用户定期备份
- 需处理数值输入验证（价格、数量必须为正数）
- 确保日期格式化正确
- 统计数据需按日期筛选
- 库存数量不能为负数
